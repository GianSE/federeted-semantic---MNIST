% ─── RESUMO ────────────────────────────────────────────────────────────────
\begin{abstract}
Este trabalho avalia se um autoencoder treinado de forma distribuída consegue
comprimir imagens sem perder seu significado semântico.
Utilizamos um Variational Autoencoder (VAE) convolucional, treinado via
Aprendizado Federado (FedAvg) entre três dispositivos simulados em contêineres
Docker, para comprimir imagens do dataset MNIST.
O resultado principal: a imagem é comprimida de \textbf{3.136 bytes para apenas
36 bytes} (redução de 98,9\% na largura de banda), e ainda assim um classificador
independente acerta 86\% das imagens reconstruídas, contra 90\% das originais —
uma queda de apenas 4\%, confirmando que a semântica é preservada.
\end{abstract}

\begin{IEEEkeywords}
Aprendizado Federado, Comunicação Semântica, Autoencoder, Compressão de Imagens, Docker
\end{IEEEkeywords}


% ─── 1. INTRODUÇÃO ────────────────────────────────────────────────────────
\section{Introdução}

Quando um dispositivo precisa enviar uma imagem pela rede, a abordagem
tradicional transmite todos os pixels — mesmo que a maior parte da informação
seja redundante. Para uma imagem MNIST de 28$\times$28 pixels, isso significa
3.136 bytes por imagem. Em cenários com muitos dispositivos IoT ou redes 5G/6G
congestionadas \cite{kairouz2021advances}, esse custo se torna um gargalo.

A \textbf{comunicação semântica} propõe uma alternativa: enviar apenas
a \textit{essência} da imagem — uma representação compacta que o receptor
usa para reconstruir o conteúdo \cite{yang2022semantic, xie2021deep}.
Um autoencoder neural aprende a comprimir a imagem em um vetor pequeno
(o ``espaço latente'') e a reconstruí-la do outro lado. O transmissor envia
o vetor comprimido; o receptor reconstrói a imagem.

O desafio é: \textbf{a imagem reconstruída ainda tem o mesmo significado?}
Um classificador consegue identificar o dígito corretamente, mesmo após
compressão extrema, ruído de canal e perdas de pacotes?

Para treinar esse autoencoder sem centralizar os dados dos dispositivos,
usamos \textbf{Aprendizado Federado} \cite{mcmahan2017}: cada dispositivo
treina localmente e compartilha apenas os pesos do modelo, nunca os dados brutos.

Este trabalho implementa e valida esse sistema em um ambiente Docker
reprodutível, respondendo à pergunta: é possível economizar 98,9\% de banda
e ainda preservar a semântica das imagens?


% ─── 2. SISTEMA ────────────────────────────────────────────────────────────
\section{Sistema Proposto}

\subsection{Como Funciona}

O pipeline de comunicação semântica tem cinco etapas simples, ilustradas
na Figura~\ref{fig:canal}:

\begin{enumerate}[leftmargin=*, label=\arabic{*}.]
    \item \textbf{Codificação}: o transmissor passa a imagem pelo \textit{encoder},
    obtendo um vetor de 32 números (o espaço latente);
    \item \textbf{Quantização}: os 32 valores float32 são convertidos para int8,
    reduzindo de 128 bytes para \textbf{36 bytes} a transmitir;
    \item \textbf{Canal}: o vetor passa por ruído gaussiano (AWGN, SNR = 15~dB)
    e masking de 10\% dos pixels — simulando condições reais de rede sem fio;
    \item \textbf{Decodificação}: o receptor usa o \textit{decoder} para reconstruir
    a imagem 28$\times$28 a partir dos 36 bytes recebidos;
    \item \textbf{Classificação}: um classificador independente avalia se o
    dígito ainda é reconhecível na imagem reconstruída.
\end{enumerate}

\begin{figure}[ht]
\centering
\includegraphics[width=\linewidth]{figures/fig_canal_semantico.png}
\caption{Canal semântico em ação: imagem original (esq.), após ruído e masking
(centro), e reconstruída pelo decoder (dir.). Transmitido: 36 bytes em vez de 3.136.}
\label{fig:canal}
\end{figure}

\subsection{O Modelo: CNN-VAE}

O autoencoder utilizado é um \textit{Variational Autoencoder} (VAE)
convolucional \cite{kingma2014vae}. A diferença de um VAE para um
autoencoder simples é que ele não produz um vetor fixo, mas aprende
uma \textit{distribuição} no espaço latente — o que torna a representação
mais regular e robusta a perturbações de canal.

A função de perda do VAE combina duas metas:
\begin{equation}
\mathcal{L} = \underbrace{\mathcal{L}_{\text{MSE}}}_{\text{fidelidade}} +
              \underbrace{D_{\text{KL}}}_{\text{regularidade do espaço latente}}
\end{equation}

O termo MSE faz a imagem reconstruída ser parecida com a original;
o termo KL organiza o espaço latente de forma contínua e regular,
o que é benéfico quando o vetor é corrompido pelo ruído do canal.

\subsection{Treinamento Federado}

O VAE é treinado por 3 contêineres Docker (clientes) sem compartilhar dados,
seguindo o algoritmo FedAvg \cite{mcmahan2017}:

\begin{enumerate}[leftmargin=*, label=\arabic{*}.]
    \item O servidor distribui os pesos iniciais;
    \item Cada cliente treina localmente por 5 épocas;
    \item Os clientes enviam seus pesos ao servidor;
    \item O servidor calcula a média dos pesos (FedAvg) e redistribui;
    \item Repete por 5 rounds.
\end{enumerate}

Todo o ambiente roda via Docker Compose, sem dependências de ML no computador
host: datasets, pesos e logs ficam em um volume compartilhado.

\begin{table}[ht]
\centering
\caption{Configuração dos experimentos}
\label{tab:params}
\begin{tabular}{ll}
\toprule
\textbf{Parâmetro}       & \textbf{Valor} \\
\midrule
Dataset                  & MNIST (28$\times$28, escala de cinza) \\
Dimensão latente         & 32 valores \\
Quantização              & float32 $\rightarrow$ int8 \\
Bytes transmitidos       & 36 B por imagem \\
Rounds FL / Épocas       & 5 rounds $\times$ 5 épocas \\

Otimizador               & Adam (lr = 0,001) \\
Canal AWGN               & SNR = 15 dB \\
Masking                  & 10\% dos pixels zerados \\
\bottomrule
\end{tabular}
\end{table}

% ─── 4. DISCUSSÃO ─────────────────────────────────────────────────────────
\section{Discussão}

Os resultados respondem diretamente à hipótese do trabalho:

\begin{itemize}[leftmargin=*]
    \item \textbf{Sim, é possível comprimir 87,1$\times$} — de 3.136 bytes para
    36 bytes por imagem — usando um VAE treinado federadamente;
    \item \textbf{Sim, a semântica é preservada} — a queda de 4,0\% na acurácia
    do classificador está dentro do critério de 5\% estabelecido;
    \item \textbf{O treinamento federado funciona} — os 3 clientes convergem
    sem compartilhar dados, loss final de 0,01038 em apenas 5 rounds.
\end{itemize}

O insight mais prático da Figura~\ref{fig:galeria} é a existência de um
\textbf{limiar de PSNR}: imagens reconstruídas com PSNR $\geq$ 18~dB são
classificadas corretamente em praticamente todos os casos. Isso sugere
que ajustar o SNR do canal latente para manter esse limiar é suficiente
para garantir a preservação semântica.

\textbf{Limitação principal:} o dataset MNIST é simples (imagens em escala de
cinza, 10 classes). Para imagens coloridas ou datasets mais complexos (CIFAR-10),
espera-se que seja necessário aumentar a dimensão latente ou a profundidade
do modelo.


% ─── 5. CONCLUSÃO ─────────────────────────────────────────────────────────
\section{Conclusão}

Este trabalho demonstrou que é viável usar comunicação semântica com
Aprendizado Federado para reduzir drasticamente a largura de banda em
sistemas distribuídos. O CNN-VAE treinado via FedAvg em 3 contêineres
Docker alcançou:

\begin{itemize}[leftmargin=*]
    \item \textbf{98,9\% de redução de banda} (3.136 B $\rightarrow$ 36 B por imagem);
    \item \textbf{PSNR = 20,61~dB e SSIM = 0,911} — reconstrução de alta qualidade;
    \item \textbf{Queda semântica = 4,0\%} — hipótese confirmada (critério $\leq$ 5\%).
\end{itemize}

Para trabalhos futuros: extensão para CIFAR-10, avaliação com canais de
desvanecimento (Rayleigh), e ajuste dinâmico da dimensão latente conforme
a qualidade do canal.
